# 貢献ガイド（CONTRIBUTING）

本プロジェクトは、小1〜高3の「デジタル・AI人材育成」教科書仕様をオープンに設計する社会貢献型 OSS です。教育関係者・エンジニア・研究者を問わず、Issue / Pull Request での貢献を歓迎します。

## とくに歓迎する貢献

| 種類 | 例 | 入口 |
|---|---|---|
| **現場知見** | 「この学年でこの誤概念が実際に多い」「この活動は時数内に収まらない」等、授業実践に基づく指摘 | Issue（現場知見テンプレート） |
| **一次情報の更新** | 学習指導要領・生成AIガイドライン・関連法令の改訂の検知（仕様書第18章の保守サイクルの重要な入力源） | Issue（一次情報テンプレート） |
| **仕様の矛盾・誤りの指摘** | 目標IDの参照ミス、章間の記述不一致、集計値の齟齬 | Issue（矛盾報告テンプレート）または PR |
| **ツール・検証の改善** | `tools/` の抽出・検証スクリプト、CI、可視化の改善 | PR |

現職教員・教科専門家のレビューは特に歓迎します（各学年の「誤概念トップ3」等、現場知見が品質を左右します）。

## 貢献の流れ

1. **Issue から始める**（推奨）: 大きな変更は着手前に Issue で方向性を確認してください。誤字修正などの軽微な変更は直接 PR で構いません
2. **ブランチを切って変更**: `main` から作業ブランチを作成します
3. **PR を作成**: 変更の目的・根拠（一次情報があれば出典）を本文に書いてください
4. **審査**: PR は仕様書第17章の品質ゲート基準に照らして審査されます。カリキュラム内容（到達目標・単元設計）に関わる変更は監修判断が必要です

## 仕様書を変更するときのルール

- 正典は `docs/specification.md` の1ファイルです。要件レベルは MUST / SHOULD / MAY で明記します（第0章）
- **到達目標・単元表を変更した場合は、目標データベースの再生成が必須です:**

  ```bash
  python3 tools/extract_goals.py   # data/goals.json を再生成（手編集しない）
  python3 tools/validate_goals.py  # 付録A.1規則 + 9.4集計の照合
  ```

  CI（`validate-spec`）が「仕様書と goals.json の同期」「検証規則」を全 push / PR で自動照合し、乖離があると失敗します
- 到達目標の記述は第3.3節の規則（行動動詞＋対象＋条件＋水準、ID体系、付録Bの動詞リスト）に従ってください
- 目標数・★数を変更する場合は 9.4 の機械集計値も更新してください（validate が不一致を検出します）
- 内容を改訂したら文書末尾の改訂履歴に1行追記してください（版番号の上げ方は保守サイクル＝第18章に従う）

## ツール（`tools/`）を変更するときのルール

- Python 3 標準ライブラリのみを使用します（依存追加はまず Issue で相談）
- 変更後は上記2コマンドがローカルで通ることを確認してください

## 行動規範・その他

- 建設的で敬意ある議論をお願いします。教育現場・児童生徒への配慮を最優先にしてください
- 児童生徒・教員など実在の個人が特定できる情報（実名・学校名・事例の詳細）を Issue / PR に書かないでください。現場知見は個人が特定されない形に一般化して共有してください
