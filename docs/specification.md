# デジタル・AI人材育成 学校向け教科書シリーズ 制作仕様書

**対象:** 小学校1年生〜高等学校3年生（全12学年・全12巻＋教師用指導書12巻＋デジタル教材）
**版:** 第2.1版（2026年7月3日改訂）
**文書の性格:** 本仕様書は、どの出版社・どの編集者が制作を担っても同水準の教育品質が再現されることを目的とした「制作の憲法」であり、本書一冊で企画から品質保証までの全工程を規定する唯一の正典である。特記なき要件はすべて必須要件（MUST）とする。推奨要件はSHOULD、任意要件はMAYと明記する。

## 第1章 目的・理念・基本方針

### 1.1 シリーズの使命

本シリーズは、日本のすべての児童生徒が、AIとデジタル技術が社会基盤となった時代において、**「技術を理解し、使いこなし、批判的に評価し、創造し、社会に責任を持って実装できる市民」** へと成長することを支援する。単なる操作スキルの習得ではなく、原理の理解・倫理的判断・創造的活用を三位一体で育成する。

### 1.2 教育理念（5原則）

1. **人間中心（Human-Centered）:** AIは人間の能力を拡張する道具であり、判断の主体は常に人間である。この原則を全学年・全単元の通奏低音とする。
2. **原理主義（Principles over Products）:** 特定製品の操作手順ではなく、製品が変わっても通用する概念・原理を教える。製品名・画面例は「差し替え可能な例示」として設計する（第14章）。
3. **創り手の育成（Creator, not just Consumer）:** 消費者としての利用にとどまらず、プログラム・データ分析・AIシステムの設計者としての経験を全学年で保障する。
4. **スパイラル構造（Spiral Curriculum）:** 同一の中核概念を発達段階に応じて繰り返し、深度を上げて再訪する。12年間で概念が断絶しないことを設計上保証する。
5. **公平性（Equity）:** 家庭のICT環境・地域・性別・障害の有無・日本語習熟度によらず学べる教材とする。アンプラグド（機器不要）代替活動を小中の全単元に用意する。

### 1.3 基本方針

- **MUST:** 次期学習指導要領（2026年度答申予定）の方向性に整合し、かつ現行学習指導要領下でも使用可能な二重適合設計とする
- **MUST:** 「教科書『を』教える」のではなく「教科書『で』教える」構造とし、各単元に学校裁量の拡張・省略ポイントを明示する
- **MUST:** 12年間の到達目標を単一の能力マトリクス（第3章）から演繹的に導出し、学年間の重複・欠落・飛躍をトレーサビリティ表で検証可能にする
- **SHOULD:** 探究的な学び（プロジェクト型学習）を各学年の年間学習時間の30〜50%に配分する

## 第3章 育成する資質・能力の全体構造

### 3.1 5領域モデル

12年間の学習内容を以下の5領域に構造化する。全学年・全単元は必ずいずれかの領域（主領域）に属し、副領域を最大2つ持てる。

| 領域 | 名称 | 中核概念 |
|---|---|---|
| **A** | コンピューティングとプログラミング | アルゴリズム、抽象化、分解、データ構造、コンピュータ・ネットワークの仕組み |
| **B** | データとAI | データの収集・整理・可視化・分析、統計的推論、機械学習の原理、生成AIの仕組みと限界 |
| **C** | 情報デザインとコミュニケーション | 情報の構造化、メディア表現、UI/UX、伝達設計、コンテンツ制作 |
| **D** | デジタル・シティズンシップと倫理 | 安全・健康、プライバシー、著作権、情報の信頼性評価、AI倫理、法とガバナンス |
| **E** | 探究と社会実装 | 課題発見、プロジェクト設計、プロトタイピング、評価・改善、社会への発信 |

### 3.2 3観点（学習指導要領の資質・能力三本柱に対応）

各単元の到達目標は必ず以下の3観点すべてで記述する（MUST）。

1. **知識・技能:** 「〜を説明できる」「〜を操作できる」
2. **思考・判断・表現:** 「〜を比較して選択できる」「〜を設計できる」「〜を評価できる」
3. **学びに向かう力・人間性:** 「〜しようとする」「〜の立場を考慮する」

### 3.3 到達目標の記述規則（再現性の要）

- **MUST:** すべての到達目標は「行動動詞＋対象＋条件＋水準」の形式で記述する
  - 例: 「（条件）与えられた身近な問題について、（対象）手順を、（行動）順次・分岐・反復を用いて図示（水準）でき、他者に説明できる」
- **MUST:** 行動動詞は付録Bの「認知レベル別動詞リスト」（改訂ブルーム・タキソノミー準拠）からのみ選択する
- **MUST:** 到達目標IDを付番する。形式: `[学年][領域][連番]`（例: `E3-B-02` ＝ 小3・データとAI・第2目標）。全巻横断のトレーサビリティ表（付録A）で管理する

## 第4章 巻詳細設計の共通規則（全学年に適用・MUST）

### 学年段階の区分と発達的前提

| 段階 | 学年 | 発達的前提と設計方針 |
|---|---|---|
| 基礎期 | 小1-2 | 文字の読み書きが発達途上。アンプラグド活動中心、写真・イラスト主体、1活動15分単位。生成AIの児童による直接利用は行わない（教師の実演のみ） |
| 体験期 | 小3-4 | ローマ字学習と並行してキーボード入力を導入。ビジュアルプログラミング開始。協働作業の作法を学ぶ |
| 活用期 | 小5-6 | 抽象的思考の芽生え。データ活用・情報発信のプロジェクトを実施。生成AIは教師の管理下・保護者同意のもと限定的に体験 |
| 展開期 | 中1-3 | 形式的操作期。テキストプログラミング、計測・制御、統計的問題解決、生成AIの主体的かつ批判的な活用へ移行 |
| 統合期 | 高1-3 | 情報Ⅰ相当を全員必修水準で、情報Ⅱ相当＋AI応用を選択発展として提供。進路（文理問わず）と接続 |

### 設計規則R1〜R9

**R1 単元表の記法:** 目標IDは `[学年]-[領域]-[連番]`。目標文は〔条件〕〔対象〕〔行動〕〔水準〕を含む一文。観点は【知】知識・技能／【思】思考・判断・表現／【学】学びに向かう力。
**R2 時数配当の根拠規則:** 各単元の時数は「導入1 : 習得2 : 活用・振り返り2」の比を基準に、実証授業（品質ゲートG3）の実測所要時間で確定する。1単位時間＝小45分／中50分／高50分。総時数の10%を予備時数（未消化・深掘り用）として単元に割り付けない。
**R3 単元間依存:** 各単元は前提目標IDを明記。前提は必ず過去に向ける（同学年内なら先行単元、学年跨ぎなら過去の学年の目標。未来の学年の目標を前提にしない）。学年差3以上の参照は付録A.3に件数を開示し、当該単元冒頭の接続欄（3.4）で前提の再活性化（既習事項の想起活動）を設計する。
**R4 評価対応:** 各目標に評価方法（確認問題／実技チェック／パフォーマンス課題PT／ポートフォリオPF／観察）を1つ以上割付。割付なき目標はG1ゲート不合格。
**R5 差異化3水準:** 全単元に(a)基礎ルート（支援が必要な学習者向けの縮約手順・具体物）、(b)標準ルート、(c)発展ルートを設計。基礎ルートでもコア目標（★印）には到達できる構造とする。
**R6 コア目標（★）:** 各学年の目標のうち、次学年の前提となるものを★指定。★目標の実証授業達成率は80%以上（非★は70%以上）をG3通過基準とする。
**R7 生成AI取り扱い学年規程:** 小1〜4＝児童の直接操作なし（教師実演・出力教材化のみ）／小5〜6＝教師管理下・保護者同意・教育用サンドボックスのみ／中1以上＝年齢要件と校内規程に基づく主体的利用＋AI利用申告。毎版、法令適合レビューで年齢要件を再確認。
**R8 教科横断割付:** 小・中の実施時数割付表は「標準割付」であり、学校は総時数を維持したまま組み替え可能と教科書・指導書に明記する（調整授業時数制度への適合）。

**R9 時数確定プロトコル**

R9-1 暫定値の許容変動幅

- 各単元の確定時数は暫定値の±20%の範囲内でのみ変更できる（例: 4時間→3.2〜4.8→実運用は3〜5時間）
- 範囲を超える実測となった単元は「時数不足」ではなく「設計不良」と判定し、G3不合格として活動を再設計する

R9-2 実測手続き

1. 検証3校（都市部・地方・特別支援学級を含む）で同一単元を実施し、活動完了までの実時間を記録
2. 3校の中央値を実測値とする（平均は外れ値に弱いため用いない）
3. 実測値が許容幅内→確定。超過→R9-3の削減規則を適用して再実測

R9-3 超過時の削減優先順位（この順で削り、★目標の活動は削らない）

1. 発展ルート活動の紙面外（デジタル教材）への移動
2. 導入フックの短縮（2発問→1発問）
3. 非★目標の活動の統合
4. それでも収まらない場合のみ単元分割を監修委員会で審議

R9-4 予備時数の運用規則

- 各学年の予備時数（総時数の約10%）は「未消化の回復」に最優先で充当し、余剰時のみ発展活動に使う
- 予備時数の消費実績はG3記録に含め、次版の暫定値改訂の入力とする

## 付録H 実証授業プロトコル

### H.1 目的
各単元の学習仮説（活動→目標到達）の検証、時数の実測確定（第4章R9）、つまずき・誤概念データの収集。

### H.2 実施設計
- 検証校: 各学年3校以上（①1人1台＋高速回線の都市部校、②回線制約のある地方校、③特別支援学級を含む学校）
- 実施者: 当該教科の免許・専門を持たない教員1名以上を必ず含める（「誰が教えても」の検証のため）
- 実施単位: 単元まるごと（部分抜粋での検証は不可）

### H.3 測定
- 事前・事後テスト: ★目標に対応する確認課題（同型異問）。到達判定はルーブリックB水準
- 観察票: つまずき発生箇所（ページ・活動番号）／教師の追加説明の発生点／所要実時間（活動別）
- 児童生徒振り返り: 難易度自己評価（3件法）＋自由記述

### H.4 判定・補正規則
- ★目標達成率80%以上・非★70%以上→合格。未達単元は原因分類（説明不足/活動設計/時数/前提未習）を行い、該当部を改稿して再実証
- 時数は3校中央値で確定（±20%規則・削減優先順位は第4章R9-3）
- 収集データは匿名化し、判定記録とともに品質記録（付録F）に保存。次版改訂の一次資料とする

### H.5 データ取り扱い
- 個人が特定できる記録は校内で匿名化してから提出／保護者への実証協力説明文書の雛形を本付録に付す／データ保持は改訂2サイクル分（約20年）を上限とする
