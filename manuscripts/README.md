# manuscripts/ — 単元原稿

執筆計画書（`docs/writing-plan.md`）第6章のフェーズに従って執筆する単元原稿を置く。原稿は組版前のテキスト設計図であり、レイアウト・ルビ・図版の実制作は組版・デザイン工程（仕様書第13章・付録D）で行う。

## 運用規則

- **1単元＝1ファイル。** ファイル名は目標IDの学年コードに合わせる: `E3-U2.md`（小3・単元2）、`J1-U7.md`（中1・単元7）、`H1-C11.md`（高1・第11章）
- **テンプレート:** すべての原稿は `TEMPLATE.md`（仕様書第11章の9部構成）から開始する
- **時数・紙面・目標は執筆計画書とトレース:** 冒頭メタ情報の執筆基準時数・紙面予算は執筆計画書2.3、到達目標は仕様書第5〜9章から転記する。原稿側で数値・目標文を変更してはならない
- **セルフチェック:** 提出前に付録Cのうち執筆者セルフチェック対象（C-1到達目標・単元構造／C-2文章・表記／C-4技術的正確性）を点検し、原稿末尾の記録欄に結果を残す。全120項目の判定はG2で行う
- **ステータス:** 冒頭メタ情報の `状態` を「草稿→セルフチェック済→G2提出→G2通過」で更新する
- **図版:** 原稿では図版指示（内容・意図・代替テキスト原稿）をテキストで書く。C-038（全図版に代替テキスト）を原稿段階で満たす
- **ルビ・配当漢字:** 原稿は通常表記で書き、学年配当漢字・ふりがなの機械検査（KPI: 適合率98%以上）は組版前工程で実施する。ただし低・中学年の文長上限（1文60字以内）は原稿段階で守る
