# instructor-guides/ — 教師用指導書

仕様書第16章の必置7項目に基づき、`manuscripts/` の教科書原稿に対応する教師用指導書を巻単位で置く。

## 運用規則

- **1巻＝1ファイル。** ファイル名は学年コードに合わせる: `E1.md`（小1）、`J1.md`（中1）、`H1.md`（高1）
- **執筆計画書との関係:** 執筆計画書（`docs/writing-plan.md`）第4章の工数モデルでは、指導書工数＝対応する教科書巻の工数の0.6倍と見積もっている。教師用指導書執筆者は教科書執筆者と分離する（第17章「誰が教えても」の検証のため）
- **必置7項目（仕様書第16章）:**
  1. 単元別指導案（本時案・板書計画・発問例・つまずき予測と手立て）
  2. 誤概念カタログ（各単元3件以上・教材原稿の「誤概念トップ3」を指導書向けに詳細化）
  3. 教員の予備知識ブリーフ（情報科免許を持たない教員向け、小中は特に厚く）
  4. 環境別実施ガイド（フル環境／最小環境／アンプラグドの3水準）
  5. 評価事例集（ルーブリック各水準の児童生徒作品例・匿名加工済み）
  6. 家庭・地域連携（保護者説明文例・外部人材活用の手引き）
  7. 研修用資料（校内研修90分×2回分のパッケージ）
- **評価事例集についての留保:** 項目5（児童生徒作品例）は、G3実証授業で得られる実際の学習者の成果物を匿名加工したものでなければならない（仕様書12.2 G3参照）。実証授業前の現時点では実データが存在しないため、本ディレクトリの各指導書では項目5を「G3実証待ち」として空欄のまま明示し、architecture（想定する水準の記述形式）のみ用意する。実在しない児童の作品例を創作して掲載することはしない
- **ステータス:** 冒頭メタ情報の `状態` を「草稿→セルフチェック済→G2提出→G2通過」で更新する（教科書原稿と同一のG2ゲートで審査）
